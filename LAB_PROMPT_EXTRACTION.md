# LAB FRQ Prompt Extraction

This document contains the complete prompt, all supporting context, and configuration
needed to reproduce the same College Board-aligned LAB FRQ questions generated by
this app in any external application.

---

## Overview

The app generates **AP Physics C: Mechanics** Free Response Questions of type
**Experimental Design and Analysis (LAB)**. Questions are generated via the
**Google Gemini API** using structured JSON output.

---

## Model & API Configuration

| Setting | Value |
|---|---|
| API | Google Gemini (`@google/genai`) |
| Model | `gemini-3-pro-preview` |
| Response format | `application/json` (structured output) |
| Thinking budget | `1024` tokens |

The model is called with `thinkingConfig: { thinkingBudget: 1024 }` enabled,
which allows the model to reason before producing its final answer — this is key
to the high quality output.

---

## Point Total

LAB questions are worth **10 points** total.

| FRQ Type | Points |
|---|---|
| Mathematical Routines (MR) | 10 |
| Translation Between Representations (TBR) | 12 |
| **Experimental Design and Analysis (LAB)** | **10** |
| Qualitative/Quantitative Translation (QQT) | 8 |

## The Full Prompt (Parameterized)

The prompt below is what gets sent to the model. Replace the placeholders in
`{{ }}` with real values before use.

**Parameters:**
- `{{UNIT}}` — The AP Physics C unit name (see Unit List below)
- `{{SELECTED_SUB_TOPICS}}` — Comma-separated list of sub-topic IDs to include (see Sub-Topic List below)
- `{{EXCLUDED_SUB_TOPICS}}` — Comma-separated list of sub-topic IDs to exclude (all sub-topics in the unit that were NOT selected)
- `{{TOTAL_POINTS}}` — Always `10` for LAB

**Sub-topic instruction logic:**
- If there ARE excluded sub-topics: use the IMPORTANT EXCLUDE instruction below
- If ALL sub-topics are selected (nothing excluded): use "You may include any relevant sub-topics within this unit."

---

```
You are an expert AP Physics C: Mechanics exam writer.
Create a unique, college-level Free Response Question (FRQ) of type "Experimental Design and Analysis (LAB)" for "{{UNIT}}".
IMPORTANT: EXCLUDE the following sub-topics from the question content: {{EXCLUDED_SUB_TOPICS}}. Focus ONLY on the selected sub-topics.

Adhere strictly to the College Board style for AP Physics C: Mechanics.

TOTAL POINTS FOR THIS QUESTION: {{TOTAL_POINTS}}.
Distribute these points reasonably among the parts (e.g., 3-5 parts).

Specifics for Experimental Design and Analysis (LAB):
- Experimental Design (LAB): Focus on designing a procedure, analyzing data (linearization), and error analysis. ALWAYS include a data table for students to analyze.

IMPORTANT FORMATTING RULES:
- **Math:** Use LaTeX for ALL mathematical expressions, variables, and equations. Wrap them in single dollar signs (e.g., $F_{net} = ma$).
- **Tables:** Use standard Markdown table syntax (e.g., | x | y | ...). Ensure columns are aligned.
- **Scoring Guide:** Format with extensive whitespace for readability. Follow this EXACT structure:

  Example Scoring Guide Format:

  ---

  ### Part (a) [3 points]

  **1 point** — For starting with the integral definition of work:

  $W = \int F(x) dx$


  **1 point** — For substituting the force function and correct limits:

  $W = \int_0^D (kx^3) dx$


  **1 point** — For evaluating the integral and arriving at the correct expression:

  $W = \frac{kD^4}{4}$


  ---

  ### Part (b) [2 points]

  **1 point** — For identifying the relationship...

  KEY FORMATTING REQUIREMENTS:
  - Use THREE blank lines between each scoring criterion
  - Use horizontal rules (---) to separate parts
  - Format point values as bold: **1 point** — Description
  - Put mathematical expressions on their own lines with blank lines above and below
  - Add extra blank line before each new Part heading
  - When showing visual aids (graphs, data tables with calculations, Free Body Diagrams) in scoring guide, reference them as "See Figure X" and provide detailed image prompts

STRUCTURE REQUIREMENTS (JSON):
- "usedSubTopics": An array of strings containing the IDs (e.g. "1.1", "1.2") of the sub-topics that were *actually* tested or relevant in this specific question.
- "questionText": The setup/scenario description ONLY. Do NOT include the sub-questions (a, b, c) here. If a table of data is needed (especially for LAB), include it here using Markdown.
- "parts": An array of objects for the sub-questions. YOU MUST GENERATE 3-5 DISTINCT PARTS.
  - "label": "Part (a)", "Part (b)", etc.
  - "text": The specific instruction (e.g. "Derive an expression for...", "Calculate the...", "Sketch a graph...").
  - "points": Integer value. The sum of all points MUST equal {{TOTAL_POINTS}}.
- "scoringGuide": The detailed rubric explaining how points are awarded for each part. Follow the formatting rules above strictly to ensure readability.
- "imagePrompts": An array of strings. Describe 1-2 essential diagrams or graphs needed for the question setup. Be extremely descriptive for an AI image generator (e.g., "A schematic line art diagram of a block on an incline plane with angle theta, simple black lines on white background, no shading").
- "scoringGuideImagePrompts": An array of strings for visual aids in the scoring guide. Include when needed for:
  • Linearized data graphs (e.g., "Graph with x-axis labeled 'm (kg)' from 0 to 0.5, y-axis labeled '1/vf (s/m)' from 0.6 to 1.4, showing 5 data points in linear trend with best-fit line, grid lines visible")
  • Extended data tables (e.g., "Table with 3 columns: 'm (kg)', 'vf (m/s)', '1/vf (s/m)'. Five rows of data showing mass values 0.10-0.50 kg, corresponding velocities, and calculated reciprocals")
  • Free Body Diagrams for solutions (e.g., "Free body diagram showing block on surface with normal force N pointing up, weight mg pointing down, friction force f pointing left, applied force F at angle theta above horizontal pointing right. All forces labeled clearly")
  Use the same detailed, technical style as question image prompts.

Ensure the scenario in "questionText" provides all necessary constants and variables (e.g., "mass M", "length L") that are referenced in the "parts".
Ensure "scoringGuide" perfectly aligns with the generated "parts".

Format the output as JSON.
```

## JSON Response Schema

The model is constrained to return this exact JSON structure:

```json
{
  "usedSubTopics": ["string"],
  "questionText": "string",
  "parts": [
    {
      "label": "string",
      "text": "string",
      "points": "number"
    }
  ],
  "imagePrompts": ["string"],
  "scoringGuide": "string",
  "scoringGuideImagePrompts": ["string"]
}
```

**Required fields:** `questionText`, `parts`, `scoringGuide`

## Image Generation (Optional but Used in App)

After the JSON is returned, the app generates diagrams using `gemini-3-pro-image-preview`
for both `imagePrompts` (question setup diagrams) and `scoringGuideImagePrompts` (scoring guide visuals).

Each image prompt is prefixed with this style wrapper before being sent:

**For question images:**
```
Technical textbook diagram, schematic line art, black and white, high contrast, white background, no shading, vector style. Clearly labeled. Subject: {{IMAGE_PROMPT}}
```

**For scoring guide images:**
```
Technical textbook diagram, schematic line art, black and white, high contrast, white background, no shading, vector style. Clearly labeled with all values and units shown. Subject: {{IMAGE_PROMPT}}
```

Image config: `imageSize: "1K"`, `aspectRatio: "16:9"`

## Key Quality Factors (Summary)

These are the specific design choices that produce high-quality, College Board-aligned output:

1. **Thinking model** — `thinkingBudget: 1024` lets the model reason before writing
2. **LAB-specific instruction** — Explicitly demands linearization, data tables, and error analysis
3. **Strict formatting rules** — LaTeX math, Markdown tables, verbose whitespace in scoring guide
4. **Structured JSON schema** — Enforces correct output shape; model cannot deviate
5. **Sub-topic exclusion** — Prevents the model from drifting into untested topics
6. **Point-total enforcement** — The sum of all part points must equal 10; stated explicitly
7. **Scoring guide format example** — An inline example in the prompt shows the exact rubric style
8. **Image prompt instructions** — Highly descriptive, "technical textbook diagram" style ensures clean physics diagrams

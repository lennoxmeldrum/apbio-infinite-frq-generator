# AP Biology FRQ Prompt Extraction

This document contains the complete prompt, all supporting context, and configuration
needed to reproduce the same College Board-aligned AP Biology FRQs generated by this
app in any external application.

AP Biology does not have a "LAB" FRQ type as AP Physics C: Mechanics does — the
redesigned (2025 CED) AP Biology exam uses **six** official FRQ types, described below.
This file retains its original filename for consistency across the FRQ-generator fleet,
but documents the AP Biology prompt pipeline end-to-end with a focus on the flagship
**Interpreting and Evaluating Experimental Results (IEE)** prompt.

---

## Overview

The app generates **AP Biology** Free Response Questions of the six official types
used on the 2025 redesigned AP Biology exam. Questions are generated via the
**Google Gemini API** using structured JSON output.

| FRQ Type (Short Code) | Official Name | Points |
|---|---|---|
| IEE | Interpreting and Evaluating Experimental Results | **9** |
| IEG | Interpreting and Evaluating Experimental Results with Graphing | **9** |
| SI  | Scientific Investigation | **4** |
| CA  | Conceptual Analysis | **4** |
| AVR | Analyze Model or Visual Representation of a Biological Concept or Process | **4** |
| AD  | Analyze Data | **4** |

Each type has a dedicated prompt builder (`buildIEEPrompt`, `buildIEGPrompt`,
`buildSIPrompt`, `buildCAPrompt`, `buildAVRPrompt`, `buildADPrompt`) in
`services/geminiService.ts`.

## Shared Prompt Header

Every type-specific prompt concatenates a shared College Board-CED context block
at its top. The shared header embeds:

- The four big ideas (Evolution, Energetics, Information Storage and Transmission,
  Systems Interactions)
- The eight units with their 2025 CED exam weightings
- The six science practices (Concept Explanation, Visual Representations, Questions
  and Methods, Representing and Describing Data, Statistical Tests and Data Analysis,
  Argumentation)
- All six official FRQ types with their part structures and point breakdowns
- The official task verbs (Calculate, Construct/Draw, Describe, Determine, Evaluate,
  Explain, Identify, Justify, Make a claim, Predict, Represent, State the null
  hypothesis, Support a claim)
- Style and notation conventions (binomial species italicisation, gene vs protein
  formatting, chi-square and Hardy–Weinberg formulae, population growth equations,
  genotype case conventions, confidence interval conventions)
- The permissible topic pool for this generation (the list of sub-topics the user
  selected, grouped by unit)
- Formatting rules for `questionText`, `parts[].text`, and `scoringGuide`

See `buildSharedHeader` in `services/geminiService.ts` for the verbatim text.
Skipping this header produces plausible-sounding but off-spec FRQs.

## The Flagship Prompt: Interpreting and Evaluating Experimental Results (IEE)

The IEE prompt is the most complete example; the other five type-specific prompts
share the same shape with different part structures. The parameters are:

**Parameters:**
- `{{TOPIC_CONTEXT}}` — The permissible topic pool as a Markdown bullet list
  grouped by unit (e.g. `- Unit 7: Natural Selection\n    - 7.2 Natural Selection\n    - 7.4 Population Genetics`)
- `{{TOTAL_POINTS}}` — Always `9` for IEE

```
You are an experienced College Board AP Biology Chief Reader / exam writer.
Produce one authentic, exam-quality FRQ of type
"Interpreting and Evaluating Experimental Results" (Question 1, 9 points)
in the STRICT 2025 CED format.

<SHARED HEADER inserted here, parameterised on {{TOPIC_CONTEXT}}>

=== FRQ TYPE: Interpreting and Evaluating Experimental Results (IEE, 9 pts) ===
This is a LONG question. Build it around an authentic experimental scenario
drawn from the permissible topic pool, accompanied by data presented as a
Markdown table and/or a figure (include a detailed imagePrompt for the
figure). Students must read, not construct, the graph.

REQUIRED PART STRUCTURE (4 parts, points MUST sum to {{TOTAL_POINTS}} = 9):

  Part A — 1 point
    Describe a biological concept, process, or model relevant to the scenario
    (e.g. what a mutation is at the DNA level, what a transmembrane protein
    does, how an enzyme interacts with its substrate). Use the task verb
    "Describe".

  Part B — 3 points
    Three sub-tasks (label them B(i), B(ii), B(iii)). Each awards 1 point.
    Focus on identifying experimental methods and/or describing data:
      - Identify the dependent variable, independent variable, or a control.
      - Identify a positive/negative control and briefly state why.
      - Justify an experimental choice the researchers made.
    Use task verbs "Identify" and/or "Justify".

  Part C — 3 points
    Three sub-tasks (C(i), C(ii), C(iii)), 1 point each. Mix of:
      - Describe a trend or significant change in the data.
      - Perform a calculation grounded in the data (allele frequency,
        growth rate, percent change, chi-square contribution, etc.).
      - Compare treatment groups using the ±2 SE̅ₓ rule or qualitative
        evidence.
    Use task verbs "Describe" and "Calculate".

  Part D — 2 points
    Two sub-tasks (D(i), D(ii)), 1 point each:
      - D(i): Predict the outcome of a related follow-up experiment.
      - D(ii): Support a claim with evidence from the data provided.
    Use task verbs "Predict" and "Support a claim" / "Provide evidence".

SCIENCE PRACTICES this question should touch: 1 (Part A), 3 (Part B),
4 and 5 (Part C), 6 (Part D).

Each part must be concrete and gradeable. Part B, C, D sub-tasks should
be listed inside the single "text" field separated by blank lines and
labeled (i), (ii), (iii).

Output as structured JSON per the schema below.

<SHARED JSON STRUCTURE REQUIREMENTS inserted here — see next section>
```

## JSON Response Schema

The model is constrained to return this exact JSON structure (enforced via
`responseSchema` in the SDK call):

```json
{
  "usedSubTopics": ["string"],
  "questionText": "string",
  "parts": [
    {
      "label": "string",
      "text": "string",
      "points": "number"
    }
  ],
  "imagePrompts": ["string"],
  "scoringGuide": "string",
  "scoringGuideImagePrompts": ["string"]
}
```

**All six fields are marked `required`** so Gemini cannot silently drop one.

After the JSON is parsed, `services/geminiService.ts` runs:

1. **`normalizeEscapedWhitespace`** over `questionText`, `parts[].text`, and
   `scoringGuide` to convert any stray literal `\n` sequences into real newlines.
2. **Post-parse validation**:
   - `parts` must be a non-empty array.
   - Sum of `parts[].points` MUST equal the declared total for the FRQ type.
   - `scoringGuide` must be a non-empty string.
   On failure, an `Error` is thrown and `App.tsx` resets the UI to SELECTION.

## Per-Type Part Structures (Summary)

Every AP Biology FRQ has exactly four parts labelled Part A, Part B, Part C, Part D.
Sub-parts are labelled (i), (ii), (iii), etc. inside the single `text` field.

| Type | A | B | C | D | Total | Includes figure? |
|---|---|---|---|---|---|---|
| IEE | 1 | 3 | 3 | 2 | 9 | Optional (apparatus / context) |
| IEG | 1 | 4 | 2 | 2 | 9 | Optional scenario schematic; **data is a TABLE**; **student constructs the graph** |
| SI  | 1 | 1 | 1 | 1 | 4 | Rare (investigation schematic only) |
| CA  | 1 | 1 | 1 | 1 | 4 | No |
| AVR | 1 | 1 | 1 | 1 | 4 | **REQUIRED** (cladogram / pathway / cycle / cross / model) |
| AD  | 1 | 1 | 1 | 1 | 4 | Optional (data in a figure or table) |

Part assessed skills (College Board alignment):
- **IEE**: Part A → SP 1, Part B → SP 3, Part C → SP 4/5, Part D → SP 6
- **IEG**: Part A → SP 1, Part B → SP 4 (graphing), Part C → SP 5/3, Part D → SP 6
- **SI**:  Part A → SP 1, Part B → SP 3, Part C → SP 3, Part D → SP 6
- **CA**:  Part A → SP 1, Part B → SP 1, Part C → SP 6, Part D → SP 6
- **AVR**: All parts → SP 2 (Visual Representations), with 1 and 6
- **AD**:  Part A/B → SP 4, Part C → SP 5/6, Part D → SP 6

## Image Generation

After the JSON is returned, the app generates diagrams using
`gemini-3-pro-image-preview` for both `imagePrompts` (question setup diagrams /
figures) and `scoringGuideImagePrompts` (scoring-guide sample graphs, annotated
model solutions).

Each image prompt is prefixed with this style wrapper before being sent:

**For question images:**
```
Biology textbook diagram, schematic line art, black ink on white background, no shading, high contrast vector style. Clearly labeled in a legible serif font. All biological structures (organelles, molecules, nodes, axes) should be clean and symmetric. Subject: {{IMAGE_PROMPT}}
```

**For scoring-guide images:**
```
Biology textbook diagram, schematic line art, black ink on white background, no shading, high contrast vector style. Clearly labeled with all values, units, and annotations shown. Subject: {{IMAGE_PROMPT}}
```

Image config: `imageSize: "1K"`, `aspectRatio: "16:9"`.

Typical AP Biology figures to request via `imagePrompts`:
- **Cladograms** with labelled tips, internal nodes, and character annotations
- **Signal transduction pathways** with ligand, receptor, cascade, and downstream effect
- **Cell-cycle diagrams** with labelled phases and checkpoints
- **Punnett squares** / pedigree charts
- **Gel electrophoresis** schematics with lanes, bands, and size ladders
- **Food webs** / trophic pyramids with labelled organisms and energy arrows
- **Protein / membrane schematics** (e.g. a transmembrane channel with labelled
  amino-acid position)
- **Experimental apparatus** (divided lake, petri dish layout, filter-paper assay)

Typical `scoringGuideImagePrompts` entries:
- A sample correctly-constructed bar graph for an IEG Part B answer key
- An annotated version of the AVR figure with the correct node for Part C
  circled

## Grading Prompt (5-Step Pipeline)

After the FRQ is generated and the student uploads their handwritten response,
`gradeSubmission` sends a single prompt containing:

1. The full `STUDENT_RESPONSE_EXTRACTION_RULES` (see `services/geminiService.ts`
   and `EXTRACTION.md`). These are biology-adapted: they cover Punnett squares,
   student-constructed graphs, pathway/cladogram annotations, gene-name
   italicisation, genotype case, 5′/3′ orientation, and the dozens of
   biology-specific symbol ambiguities.
2. A curated FRQ payload (questionText, parts, scoringGuide, maxPoints).
3. The student's scanned PDF / image as an inline data part.
4. A 5-step task:
   - **STEP 1 — EXTRACT** verbatim per-part (and per-sub-part) text from the
     submission.
   - **STEP 2 — GRADE** strictly per the scoring guide. Quote the student's
     extracted writing when explaining each point decision.
   - **STEP 3 — FEEDBACK** per-part explaining why points were earned or lost.
   - **STEP 4 — TALLY** out of `maxPoints`; never exceed it.
   - **STEP 5 — FORMAT** in Markdown with single-dollar LaTeX for formulae.

The response schema is:

```json
{
  "score": "number",
  "maxScore": "number",
  "feedback": "string",
  "breakdown": "string",
  "extractedResponse": "string"
}
```

All five fields are marked `required`; the UI renders `extractedResponse`
alongside `feedback` and `breakdown` so the marker can audit the per-part
verbatim reading that fed each point decision.

And the type-specific body of the prompt reads verbatim as the IEE prompt
above. This produces questions in the style of the real 2025 CED sample —
e.g. the *Anopheles gambiae* pyrethroid-resistance FRQ, where Part A asks

## Key Quality Factors (Summary)

These are the specific design choices that produce high-quality, College
Board-aligned output:

1. **Thinking model** — `thinkingBudget: 1024` (generation) and `2048` (grading)
   lets the model reason before writing.
2. **Per-type prompt builders** — each of the six FRQ types has a dedicated
   builder that enforces its exact part structure and point distribution.
3. **CED-injected shared header** — the 2025 CED units, weightings, science
   practices, FRQ-type part structures, and task-verb list are all named
   in-prompt so the model doesn't freelance.
4. **Strict formatting rules** — real newlines (no literal `\n`), Markdown
   tables for data, single-dollar LaTeX for the handful of biology formulae
   (chi-square, Hardy-Weinberg, population growth).
5. **Structured JSON schema** — enforces correct output shape; model cannot
   deviate.
6. **Post-parse validation** — the sum of part points must equal the declared
   total (9 or 4 depending on type); mismatches throw and reset the UI.
7. **`normalizeEscapedWhitespace`** — post-processes text fields to fix stray
   `\n` leaks before the UI renders them.
8. **AVR requires an image prompt** — the prompt builder explicitly demands a
   visual model for the type that is defined by its visual.
9. **Biology-adapted extraction rules** — 5-step grading pipeline uses
   extraction rules that handle Punnett squares, 5′/3′ orientation, genotype
   case, pathway arrow types, gene-vs-protein conventions, and the common
   mitosis/meiosis and transcription/translation confusions.
10. **Image prompts carry a biology-textbook style wrapper** — schematic,
    clean black-on-white line art with serif labels, producing figures that
    match College Board's own diagram aesthetic.
